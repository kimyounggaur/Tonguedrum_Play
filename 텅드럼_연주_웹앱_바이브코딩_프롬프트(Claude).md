# 🪘 스틸 텅드럼 연주 웹앱 — 바이브코딩 프롬프트 (FINAL)

> 키보드와 마우스/터치로 **스틸 텅드럼(steel tongue drum)** 을 연주하는 인터랙티브 웹앱 명세서입니다.
> 첨부했던 칼림바 명세서의 검증된 구조(데이터 모델 · 순수함수 · 테스트 케이스 · 접근성 · 엣지케이스 · 단계별 개발 · 체크리스트)를 그대로 계승하되,
> **텅드럼의 본질적 차이**를 반영해 다음을 바꿨습니다.
>
> 1. **원형 SVG 드럼** — 칼림바는 막대(tine)가 좌→우 직선 배열이지만, 텅드럼은 텅(tongue)이 **원형/방사형**으로 배치됩니다. 텅을 SVG로 직접 그려서 **정렬이 100% 정확**하고 **외부 이미지가 필요 없는** 자급자족 앱으로 만듭니다.
> 2. **음역 재설계** — 실제 C장조 텅드럼 규격에 맞춰 **11키 = C4~F5(1.5옥타브)**, **15키 = F3~F5(2옥타브)** 로 정의합니다.
> 3. **텅드럼 음색** — 둥근 어택 + 긴 잔향 + 약한 비배음(inharmonic) 파셜 + 합성 리버브로 명상적·종(bell) 같은 소리를 합성합니다.
> 4. **차분한 명상 톤** — 카와이 대신 젠/사운드힐링 분위기(딥 틸·인디고 배경, 따뜻한 메탈·앰버 글로우).
>
> **사용자 지정 입력 규칙**(숫자키=가운데 음 / `Shift`+숫자=높은음 C5↑ / `Ctrl`+숫자=낮은음 B3↓)은 **그대로 핵심 규격**으로 적용합니다.

## 1. 프로젝트 개요 & 핵심 요구사항

브라우저에서 **키보드와 마우스/터치로 스틸 텅드럼을 연주**하는 인터랙티브 웹앱.

- **화면 1 (선택)**: `11키 스틸 텅드럼` / `15키 스틸 텅드럼` 중 하나를 고른다. 각 카드에는 **해당 드럼의 미니 SVG 미리보기**가 보인다.
- **화면 2 (연주)**: 선택한 드럼이 **위에서 내려다본 원형**으로 화면 중앙에 크게 표시되고, 텅을 키보드/클릭/터치로 연주한다.
- **입력 규칙(핵심 ★)**: `숫자키` = 가운데 음(C4~B4) / `Shift`+`숫자` = 높은음(C5↑) / `Ctrl`+`숫자` = 낮은음(B3↓, **15키 전용**).
- **사운드**: Web Audio API **합성**(mp3 없이 동작, 추후 샘플 교체 쉽게 구조화).
- **톤**: 차분하고 명상적인(젠/사운드힐링) 분위기. 둥근 형태, 따뜻한 메탈 질감, 부드럽고 느린 마이크로 인터랙션, 은은한 잔향.

## 2. 기술 스택 & 결과물 형태

- **권장(기본): 단일 `index.html`** (HTML + CSS + Vanilla JS + 인라인 SVG, 빌드·외부 라이브러리 불필요).
  - 더블클릭으로 바로 실행, 배포·공유 간단, 이미지 의존 없음.
- 사운드: **Web Audio API** (`AudioContext`, `OscillatorNode`, `GainNode`, `BiquadFilterNode`, `ConvolverNode`).
- 그래픽: **인라인 SVG** (`viewBox`로 비율 고정 → 어떤 화면에서도 무왜곡 스케일).
- 코드 품질(단일 파일 안에서도 지킨다):
  - 음/배치 데이터는 **데이터 객체·배열로 분리**(하드코딩 분산 금지).
  - 키 입력 → 음 변환은 **테스트 가능한 순수 함수**로.
  - **접근성**(role/aria/focus) 반영.
- (선택) React + TypeScript + Vite 구조를 원하면 **부록 C** 참고. 단, 본 명세는 단일 파일 기준.

## 3. 화면/레이아웃 규격 — 반응형 풀스크린 (드럼 중앙)

앱은 **뷰포트를 가득 채우고**, 원형 드럼을 **가용 공간 중앙에 최대 크기**로 배치한다. (칼림바와 달리 9:16 고정이 아니라, 원형 드럼이 어느 화면에서도 정중앙에 크게 놓이는 **반응형**이 목표다.)

- 루트(`#app`)는 `100dvh`(폴백 `100vh`)로 화면 전체를 채운다.
- 화면 1·2는 각각 `#app` 안에서 `position:absolute; inset:0;`로 가득 채운다. 빈 흰 여백 금지(테마 그라데이션 배경으로 채움).
- **연주 화면 구성(세로 스택)**: 상단 슬림 바 → **중앙 드럼(가장 크게)** → 하단 키 안내/최근음.
  - 드럼 SVG는 **정사각형 컨테이너**에 넣고, 컨테이너 크기는 `min(가용 너비, 가용 높이)` 기준으로 잡아 **항상 원형이 화면에 꽉 차되 잘리지 않게** 한다.
  - 모바일 세로: 드럼이 화면 폭에 맞춰 크게. 데스크톱/가로: 드럼이 높이에 맞춰 중앙에 크게, 양옆은 차분한 배경으로 채움.
- 모든 크기·간격은 `%`, `vmin`, `clamp()`, `min()/max()`로 잡아 **비율이 깨지지 않게** 한다.

**레퍼런스 CSS**
```css
* { box-sizing: border-box; }
html, body { margin: 0; height: 100%; }
body {
  display: grid; place-items: center;
  min-height: 100dvh;
  font-family: "Pretendard", system-ui, sans-serif;
  background: radial-gradient(120% 120% at 50% 0%, var(--bg-2), var(--bg-1));
  color: var(--ink);
}
#app {
  position: relative;
  width: 100vw; height: 100dvh;
  overflow: hidden;
}
.screen { position: absolute; inset: 0; display: flex; flex-direction: column; }
.screen[hidden] { display: none; }

/* 연주 화면: 드럼을 정사각형으로 중앙 최대 배치 */
.play-main { flex: 1 1 auto; display: grid; place-items: center; min-height: 0; padding: 2vmin; }
.drum-frame {
  width: min(100%, 100%);
  aspect-ratio: 1 / 1;             /* 항상 정사각형(원형 드럼) */
  max-width: min(92vw, 70vh);      /* 화면에 맞춰 최대, 잘림 방지 */
  margin: auto;
}
.drum-svg { width: 100%; height: 100%; display: block; }
```

## 4. ★ 텅드럼 렌더링 — SVG 원형 드럼 (핵심 차이점)

**드럼을 위에서 내려다본 모습**으로 SVG에 직접 그린다. `viewBox="0 0 100 100"`, 중심 `(50,50)`.

**구성 요소**
1. **본체(원반)**: `<circle cx="50" cy="50" r="48">` — **메탈 질감 radial-gradient**(가장자리 어둡고 중앙 밝게)로 채워 입체감. 바깥에 얇은 림(`stroke`)과 미세한 안쪽 그림자.
2. **중앙 장식(선택)**: 작은 동심원/로고로 시선 고정(텅과 겹치지 않게).
3. **텅(tongue) N개**: 각 텅은 `<g class="tongue" role="button" tabindex="0">` 로, 안에 **둥근 텅 모양**(아래 참고)과 **슬릿(절개선) stroke**를 둔다.
   - 배치는 **12번 좌표/헬퍼**로 **원형(방사형)** 으로 둔다. **가장 낮은 음 = 가장 큰 텅 = 맨 아래(6시)**, 위로 갈수록 작고 높은 음.
   - 각 텅에는 `data-note`(예: `C4`)와 `data-key`(예: `1`, `⇧1`, `⌃4`)를 넣는다.

**텅 모양 가이드**
- 한 텅은 **중심을 향해 누운 둥근 막대(스타디움/물방울) 형태**가 텅드럼처럼 보인다. 구현은 둘 중 편한 쪽:
  - `<rect rx>` (라운드 큰 사각형)을 중심 방향으로 **회전**(`transform="rotate()"`), 또는
  - `<ellipse>` 를 방사 방향으로 길게.
- 텅을 감싸는 **슬릿**은 가는 곡선 `stroke`로 그려 "잘린 금속" 느낌을 낸다(채움은 본체보다 살짝 밝거나 어둡게).
- **저음일수록 길고 크게**(midi가 낮을수록 size↑). 시각적 위계가 곧 음 높이 안내가 된다.

**상태/효과**
- 평상시: 은은한 메탈 채움.
- **연주 시(`.is-active`)**: 텅 채움이 잠깐 밝아지고 **앰버 글로우**(`filter: drop-shadow`/외곽 발광) + **물결(ripple)** 한 번. 약 0.25~0.4s 후 원복.
- (선택) 각 텅 위에 **숫자악보/단축키 라벨**(작은 `<text>`)을 표시하고 토글로 끌 수 있게.

> 텅은 **논리적 음 위치**다. SVG로 그리므로 클릭 영역과 시각 요소가 **정확히 일치**한다(칼림바 사진 오버레이의 "대략 정렬" 문제가 없음). **키보드 연주가 1차 수단**이며 어떤 경우에도 정확히 동작해야 한다.

## 5. 공통 디자인 시스템

스틸 텅드럼의 **차분하고 명상적인(젠/사운드힐링)** 분위기를 따른다.

**색상(CSS 변수)**

| 토큰 | 값(예시) | 용도 |
|---|---|---|
| `--bg-1` | `#0E2A2E` | 배경 그라데이션(딥 틸) |
| `--bg-2` | `#13203A` | 배경 그라데이션(인디고) |
| `--drum-11` | `#B5772E` (하이라이트 `#E0AE6B`) | **11키** 드럼 메탈(따뜻한 브론즈) |
| `--drum-15` | `#3E6E78` (하이라이트 `#8FC3CC`) | **15키** 드럼 메탈(쿨 스틸틸) |
| `--rim` | `#1A1410` | 드럼 가장자리/슬릿 음영 |
| `--glow` | `#FFD27A` | 연주 시 하이라이트(따뜻한 앰버) |
| `--ink` | `#F2ECE0` | 본문 텍스트(소프트 크림) |
| `--muted` | `#9FB2B6` | 보조 텍스트 |

**스타일 원칙**: 둥근 형태와 부드러운 그림자, **느리고 차분한** 전환(0.3~0.45s 페이드/슬라이드), 연주 물결은 **부드럽게 확산 후 소멸**. `prefers-reduced-motion`이면 애니메이션 최소화. (칼림바의 톡톡 튀는 바운스보다 **잔잔한** 움직임.)

> **두 드럼을 색으로 구분**: 11키=브론즈 계열, 15키=스틸틸 계열. 선택 화면과 연주 화면에서 일관되게 적용한다.

## 6. [화면 1] 선택 화면 — 11키 / 15키 카드

**레이아웃(풀스크린)**
- 상단: 앱 타이틀(예: "🪘 스틸 텅드럼") + 한 줄 안내("연주할 텅드럼을 골라주세요").
- 본문: **카드 2개**(모바일=세로 스택, 넓은 화면=가로 2열, `flex-wrap`로 자동 전환).
  - **11키 카드**: 브론즈 **미니 SVG 드럼(텅 11개)** + 라벨 **"11키 텅드럼"** + 부제 "C4–F5 · 1.5옥타브".
  - **15키 카드**: 스틸틸 **미니 SVG 드럼(텅 15개)** + 라벨 **"15키 텅드럼"** + 부제 "F3–F5 · 2옥타브".
- 미니 드럼은 4번과 같은 SVG를 **작게** 재사용(텅 개수·색만 다름).

**상호작용**
- hover/focus: `scale(1.03)` + 떠오르는 그림자(부드럽게).
- 클릭/Enter/Space:
  1. 선택 카드 강조, 반대 카드 흐리게(`opacity:.4`).
  2. 클릭 지점 **물결(ripple)**.
  3. 0.3~0.45s 후 **연주 화면으로 페이드 전환**(선택값 `11|15` 전달).
- 접근성: 카드는 `role="button"`, `tabindex="0"`, `aria-label="11키 텅드럼 선택"`. 또렷한 포커스 링.

## 7. [화면 2] 연주 화면 — SVG 드럼 + 인터랙티브 텅

**레이아웃(풀스크린, 세로 스택)**
- **상단 바(슬림)**: 좌측 `← 다시 선택`(화면 1 복귀), 가운데 이름("11키 텅드럼"), 우측 **볼륨 슬라이더**.
- **메인(가장 크게)**: `.drum-frame`(정사각형) 안에 **선택된 SVG 드럼**을 무왜곡·최대 크기로(3·4번).
- **하단(접이식 가능)**: **키 안내 패널** — "숫자=가운데 / ⇧+숫자=높은음 / ⌃+숫자=낮은음(15키)" 요약 + 매핑 표 토글. **최근 연주음 배지**(예: "도 C4 · 1")를 잠깐 표시.

**연주 동작(키보드·클릭·터치 공통)**
- 모두 같은 `playNote(noteId)` 를 호출한다.
- 누르면 해당 텅에 **글로우 + 물결**이 잠깐 뜨고 소리 재생. **드럼 본체 형태는 변형하지 않는다**(텅만 반응).
- 접근성: 각 텅 `role="button"`, `aria-label="도, C4, 단축키 1"`.

> 텅 개수가 드럼 종류와 일치해야 한다(11키=11, 15키=15). 11키에는 낮은음 텅이 없으므로 `Ctrl`+숫자는 무음, 15키에만 낮은음 텅(F3~B3)이 존재한다.

## 8. 키보드 입력 규칙 (핵심 ★)

세 음역을 **수식키**로 구분한다. (사용자 지정 규칙 그대로)

| 입력 | 음역 | 비고 |
|---|---|---|
| `숫자키` 단독 | 가운데 옥타브 **C4~B4** | 두 드럼 공통 |
| `Shift` + `숫자키` | 높은음 **C5 이상** | 두 드럼 공통 |
| `Ctrl` + `숫자키` | 낮은음 **B3 이하** | **15키 전용**(11키는 무음) |

**설계 원칙 — "같은 숫자키 = 같은 계이름(도레미), 수식키 = 옥타브"**
> `1~7` = 도·레·미·파·솔·라·시(C·D·E·F·G·A·B). 수식키가 옥타브를 결정: 단독=가운데, `Shift`=위, `Ctrl`=아래.
> 예) **4번 키 = 항상 '파'** → `Ctrl+4=F3`, `4=F4`, `Shift+4=F5`.
> 본 텅드럼의 최고음은 **F5**이므로 **`Shift`는 1~4(C5~F5)만** 사용한다. `Shift+5~0`은 무음.
> 본 텅드럼의 최저음은 **F3**(15키)이므로 **`Ctrl`은 4~7(F3~B3)만** 사용한다. 그 외 `Ctrl` 조합은 무음.

## 9. 음 데이터 (11키 / 15키)

> **표준 C장조.** 가운데(맨 아래·가장 큰 텅)가 가장 낮은 음, 위로 갈수록 높아진다.
> - **11키** = `C4~F5` (1.5옥타브, 11음). 낮은음 없음.
> - **15키** = `F3~F5` (2옥타브, 15음). 11키 음역에 **저음 4개(F3·G3·A3·B3)** 를 더한 구조.

**주파수(Hz) & 숫자악보(자음보) — 공통 음표 테이블**
```js
const NOTE = {
  // note: [frequency(Hz), 숫자악보]   (.숫자 = 낮은옥타브, 숫자' = 높은옥타브)
  F3:[174.61,".4"], G3:[196.00,".5"], A3:[220.00,".6"], B3:[246.94,".7"],
  C4:[261.63,"1"],  D4:[293.66,"2"],  E4:[329.63,"3"],  F4:[349.23,"4"],
  G4:[392.00,"5"],  A4:[440.00,"6"],  B4:[493.88,"7"],
  C5:[523.25,"1'"], D5:[587.33,"2'"], E5:[659.25,"3'"], F5:[698.46,"4'"]
};
// 공식: freq = 440 * 2^((midi-69)/12)
// MIDI: F3=53,G3=55,A3=57,B3=59, C4=60..B4=71, C5=72,D5=74,E5=76,F5=77
```

**물리 배열 (원형 링, 맨 아래에서 시작 → 양옆을 따라 위로 올라가며 높아짐)**
```js
// 11키 (총 11개) — i=0 을 6시(맨 아래·가장 낮음)에 두고 원형으로 배치
const ORDER_11 = ["C4","E4","G4","B4","D5",   // 한쪽을 따라 위로 상승
                  "F5",                       // 맨 위(가장 높음)
                  "E5","C5","A4","F4","D4"];  // 반대쪽을 따라 아래로

// 15키 (총 15개)
const ORDER_15 = ["F3","A3","C4","E4","G4","B4","D5",  // 한쪽을 따라 상승
                  "F5",                                 // 맨 위(가장 높음)
                  "E5","C5","A4","F4","D4","B3","G3"];  // 반대쪽을 따라 아래로
```
> 위 배열은 "둘레를 도는 순서"의 **기본값**이다. 양옆 모두 위로 갈수록 음이 높아지는 **대칭형 텅드럼 배치**를 만든다. 실제 보이는 느낌이 어색하면 **12번 좌표만** 보고 시각적으로 미세조정한다(키보드 매핑은 그대로 유지).

## 10. 키 ↔ 음 매핑 완성표

**가운데 — 숫자키 단독 (공통)**: `1`=C4, `2`=D4, `3`=E4, `4`=F4, `5`=G4, `6`=A4, `7`=B4. (`8/9/0` 단독 = 무음)

**높은음 — Shift+숫자 (공통, ⇧1~4만 유효)**

| 키 | 음 | 키 | 음 |
|---|---|---|---|
| ⇧1 | C5 | ⇧3 | E5 |
| ⇧2 | D5 | ⇧4 | F5 |

> `⇧5 ~ ⇧0` = **무음**(본 텅드럼 최고음이 F5).

**낮은음 — Ctrl+숫자 (★ 15키 전용, ⌃4~7만 유효)**: `⌃4`=F3, `⌃5`=G3, `⌃6`=A3, `⌃7`=B3. (11키에선 전부 무음, `⌃1/2/3/8/9/0`도 무음)

**참고 매핑 객체**
```js
const MAP = {
  MID:  { 1:"C4", 2:"D4", 3:"E4", 4:"F4", 5:"G4", 6:"A4", 7:"B4" }, // 숫자 단독
  HIGH: { 1:"C5", 2:"D5", 3:"E5", 4:"F5" },                         // Shift (1~4)
  LOW:  { 4:"F3", 5:"G3", 6:"A3", 7:"B3" }                          // Ctrl (15키 전용)
};
```

## 11. 키 입력 처리 로직 (순수 함수 + 엣지케이스 + 테스트)

**반드시 지킬 규칙**
1. **`event.code` 사용**(`event.key` 금지). `Shift+1`은 `key`가 `"!"`로 바뀌기 때문. **`Digit1~0`과 `Numpad1~0` 둘 다** 지원.
2. **`Alt` 또는 `Meta`(⌘)** 가 눌리면 → 무음(OS/브라우저 단축키 충돌 방지).
3. **`Ctrl`+`Shift` 동시** → 무음(오입력 처리).
4. 수식키 없는 `8/9/0` → 무음. **`Shift`는 1~4만**(`⇧5~0` 무음). **`Ctrl`은 4~7만**(그 외 무음).
5. `Ctrl`+숫자는 **15키에서만** 동작(`⌃4~7`→F3~B3). 11키는 전부 무음.
6. **`event.repeat`(꾹 누름)** 무시. 단, 같은 키를 따로 빠르게 여러 번 누르면 매번 재생.
7. 매핑된 조합에서 **`event.preventDefault()`** 호출(탭 전환·저장 등 차단).
8. 포커스가 `input/textarea/select/contenteditable` 안이면 연주하지 않음.

**테스트 가능한 순수 함수(이 시그니처로 구현)**
```js
// resolveShortcut({code, shiftKey, ctrlKey, altKey, metaKey}, type:"11"|"15") -> noteId | null
function digitOf(code){
  const m = /^(?:Digit|Numpad)([0-9])$/.exec(code);
  return m ? Number(m[1]) : null;        // 0..9 | null
}
const MAP = {
  MID:  { 1:"C4",2:"D4",3:"E4",4:"F4",5:"G4",6:"A4",7:"B4" },
  HIGH: { 1:"C5",2:"D5",3:"E5",4:"F5" },
  LOW:  { 4:"F3",5:"G3",6:"A3",7:"B3" }   // 15키 전용
};
function resolveShortcut(e, type){
  if (e.altKey || e.metaKey) return null;
  if (e.shiftKey && e.ctrlKey) return null;
  const d = digitOf(e.code); if (d === null) return null;
  if (e.shiftKey) return MAP.HIGH[d] ?? null;
  if (e.ctrlKey)  return type === "15" ? (MAP.LOW[d] ?? null) : null;
  return MAP.MID[d] ?? null;
}
```

**필수 테스트 케이스**
- `Digit1` (수식X, 11) → `C4`
- `Digit7` (수식X) → `B4`
- `Digit8` (수식X) → `null`
- `Shift+Digit1` → `C5`, `Shift+Digit4` → `F5`
- `Shift+Digit5` → `null`  ← (G5 없음)
- `Shift+Digit8` → `null`  ← (C6 없음)
- `Ctrl+Digit4` (11) → `null`
- `Ctrl+Digit4` (15) → `F3`, `Ctrl+Digit7` (15) → `B3`
- `Ctrl+Digit3` (15) → `null`  ← (LOW는 4~7만)
- `Ctrl+Shift+Digit4` → `null`
- `Alt+Digit1` / `Meta+Digit1` → `null`
- `Numpad1` (수식X) → `C4`

**데이터 검증**: 11키 음 11개·15키 음 15개, `ORDER_11`·`ORDER_15`의 모든 noteId가 `NOTE`에 존재.

## 12. 텅 배치 좌표 & 헬퍼 (원형/방사형)

텅을 **원형으로 배치**하는 기본 헬퍼. SVG `viewBox 0 0 100 100`, 중심 `(50,52)` 기준. (칼림바의 "tine band N등분" 대신 텅드럼은 **각도(angle)로 배치**.)

```js
const freqOf = n => NOTE[n][0];
const midiOf = n => Math.round(69 + 12*Math.log2(freqOf(n)/440));

// order 배열의 i번째 텅을 원 둘레에 배치 + 낮은 음일수록 크게
function layoutTongues(order){
  const cx = 50, cy = 52;     // 중심(%)  — 시각적으로 미세조정 가능
  const R  = 30;              // 텅 중심이 놓이는 반지름(%)
  const N  = order.length;
  const LOW = 53, HIGH = 77;  // F3~F5 (midi)
  return order.map((note, i) => {
    // i=0 을 맨 아래(6시)에 두고 시계방향으로 균등 배치
    const ang = Math.PI/2 + (i * 2*Math.PI / N);   // 라디안(아래에서 시작)
    const x = cx + R * Math.cos(ang);
    const y = cy + R * Math.sin(ang);
    const t = (midiOf(note) - LOW) / (HIGH - LOW); // 0(낮음)~1(높음)
    const len = 26 - 10 * t;   // 텅 길이(%): 낮을수록 길게(약 26~16)
    const wid = 12 - 3 * t;    // 텅 폭(%):  낮을수록 넓게(약 12~9)
    const rotDeg = ang * 180/Math.PI + 90; // 중심을 향하도록 회전
    return { note, x, y, len, wid, rotDeg };
  });
}
```

**구현 메모**
- 위 값(`cx,cy,R,len,wid`)은 **시작점**이다. 드럼이 너무 빽빽하거나 헐거우면 **이 숫자만** 조정한다.
- 15키가 한 링에 빽빽하면 **저음 4개(F3·G3·A3·B3)를 안쪽 작은 링**, 나머지를 바깥 링으로 두는 **이중 링** 배치도 실제 15키 드럼과 유사하다(선택). 기본은 단일 링으로 깔끔하게.
- 각 텅은 `rotate(rotDeg, x, y)` 로 중심을 향하게 그린 **둥근 막대**(`<rect rx>` 또는 `<ellipse>`)로 렌더링한다.
- 텅의 `aria-label`은 `"<계이름>, <음명>, 단축키 <키표시>"` 형식(예: `"파, F3, 단축키 Ctrl+4"`).

## 13. 사운드 엔진 (Web Audio 합성 — 텅드럼 음색)

**목표 음색**: **둥근 어택 + 길게 울리는 잔향**, 약한 **비배음(metallic)** 파셜, 낮은 음일수록 더 길게. 종(bell)과 마림바 사이의 따뜻하고 명상적인 소리. **폴리포니**(동시·연속 발음). (칼림바의 톡 튀는 'pluck'보다 둥글고 'bong'에 가깝다.)

**필수 사항**
- `AudioContext`는 **1회 생성·재사용**. **첫 사용자 입력(키/클릭) 시 `ctx.resume()`**.
- **마스터 게인 + 볼륨 슬라이더**(0~1). 음마다 독립 voice 생성 후 종료 시 정리.
- 텅드럼은 **자연히 길게 울리고 사라진다** → `keyup`에서 소리를 강제로 끊지 않고 **자연 감쇠**시킨다(시각 하이라이트만 keyup/타이머로 해제).
- (권장) **합성 임펄스 리버브**로 은은한 잔향(meditative bloom)을 더한다. mp3 불필요.

**레퍼런스 구현**
```js
const ctx = new (window.AudioContext || window.webkitAudioContext)();
const master = ctx.createGain(); master.gain.value = 0.8; master.connect(ctx.destination);

// (권장) 합성 임펄스 리버브 — 명상적 잔향
const convolver = ctx.createConvolver();
convolver.buffer = makeIR(ctx, 2.6, 2.0);          // 약 2.6초 꼬리
const wet = ctx.createGain(); wet.gain.value = 0.25; // 잔향 양(0~0.5 권장)
master.connect(wet); wet.connect(convolver); convolver.connect(ctx.destination);

function makeIR(ctx, seconds, decay){
  const rate = ctx.sampleRate, len = Math.max(1, Math.floor(rate * seconds));
  const buf = ctx.createBuffer(2, len, rate);
  for (let ch = 0; ch < 2; ch++){
    const d = buf.getChannelData(ch);
    for (let i = 0; i < len; i++) d[i] = (Math.random()*2 - 1) * Math.pow(1 - i/len, decay);
  }
  return buf;
}

const setVolume = v => master.gain.value = Math.max(0, Math.min(1, v));
const freqOf = n => NOTE[n][0];

function playNote(noteName){
  const f = freqOf(noteName); if (!f) return;
  if (ctx.state === "suspended") ctx.resume();          // ★ autoplay 해제
  const now = ctx.currentTime;
  const midi = 69 + 12*Math.log2(f/440);
  const dur = Math.max(2.2, 4.6 - (midi - 53) * 0.06);  // 낮을수록 길게(약 2.2~4.6s)

  // 감쇠하며 어두워지는 로우패스(부드러운 메탈 쉬머)
  const lp = ctx.createBiquadFilter(); lp.type = "lowpass";
  lp.frequency.setValueAtTime(Math.min(8000, f*6), now);
  lp.frequency.exponentialRampToValueAtTime(Math.max(350, f*2), now + dur);

  const env = ctx.createGain(); env.connect(lp).connect(master);
  env.gain.setValueAtTime(0.0001, now);
  env.gain.exponentialRampToValueAtTime(0.85, now + 0.012); // 둥근 어택(~12ms)
  env.gain.exponentialRampToValueAtTime(0.0001, now + dur); // 긴 지수 감쇠

  // 종/금속 느낌의 약한 비배음 파셜
  [[1,1.0],[2.0,0.5],[2.76,0.26],[5.18,0.08]].forEach(([mul,g])=>{
    const o = ctx.createOscillator(); o.type = "sine"; o.frequency.value = f*mul;
    const og = ctx.createGain(); og.gain.value = g;
    o.connect(og).connect(env); o.start(now); o.stop(now + dur + 0.1);
  });
}
```
> (선택) 더 사실적인 소리는 음별 샘플(.wav/.mp3) 로드로 교체 가능하게 `playNote`만 갈아끼우면 되도록 분리한다.

## 14. 상호작용 & 시각 효과

- **연주 피드백(키보드/클릭/터치 공통)**: 해당 텅에 **앰버 글로우 + 부드러운 물결**이 잠깐 뜨고 사라짐. **드럼 본체는 변형 금지**(텅만 반응).
- `keydown`에서 소리+효과 발동, 시각 효과는 `keyup`(또는 0.3~0.4s 타이머)으로 해제. **소리는 끊지 않고 자연 감쇠**. `event.repeat` 무시.
- **최근 연주음 배지**(예: "도 (C4) · 1")를 상단/하단에 잠깐 표시 후 페이드아웃.
- (접근성) 시각 효과와 별개로 `aria-live="polite"` 영역에 최근 음 이름을 갱신.
- 움직임은 **잔잔하게**(부드러운 ease, 천천히 사라지는 물결). `prefers-reduced-motion`이면 글로우만 켜고 물결/스케일은 생략.

- 모든 클릭 요소는 키보드 포커스 가능 + 또렷한 포커스 링.

## 16. 단계별 개발 순서 (Phase 1~7)

> 각 단계 후 동작 확인하고 진행.

1. **뼈대 & 반응형 풀스크린**: `#app`(100dvh) + `.screen` 2개(선택/연주) 토글 + `← 다시 선택` + 차분한 그라데이션 배경. `.drum-frame`(정사각형 중앙) 확인.

- [ ] 드럼/텅이 **SVG로 렌더링**되어 외부 이미지 없이 동작하고, 화면 크기가 바뀌어도 비율이 유지된다.
- [ ] 텅 클릭 영역과 시각 요소가 **정확히 일치**한다(텅 자체가 버튼).

**연주/매핑**
- [ ] 11키=텅 11개(C4~F5), 15키=텅 15개(F3~F5)가 원형 배열로 배치(낮은 음=맨 아래·가장 큰 텅).
- [ ] `1~7`→C4~B4, `⇧1~4`→C5~F5, `⌃4~7`→F3~B3(**15키만**), 11키 `Ctrl` 무음.
- [ ] `같은 숫자=같은 계이름`(`⌃4=F3, 4=F4, ⇧4=F5`).
- [ ] `8/9/0` 단독·`⇧5~0`·`⌃1/2/3`·`Ctrl+Shift`·`Alt`·`Meta` 무음. `Numpad`도 동작.
- [ ] 키 꾹 눌러도 연타 안 됨, 빠른 반복은 매번 재생.
- [ ] `Ctrl+숫자`로 브라우저 탭 안 바뀜(`preventDefault`).
- [ ] 키보드·마우스·터치 모두 소리. 폴리포니. 첫 입력에서 정상 발음(`resume`). 소리는 자연 감쇠.
- [ ] 누른 텅 즉시 글로우/물결 + 최근음 배지.

**음색/품질**
- [ ] 둥근 어택 + 긴 잔향 + 약한 비배음 파셜(텅드럼다운 소리). (권장) 리버브로 은은한 잔향.
- [ ] 순수함수 테스트 케이스 통과. 데이터 개수·존재 검증 통과.
- [ ] 포커스 링/`aria-label`/`aria-live` 동작. `prefers-reduced-motion` 반영.

## 부록 A. 그대로 복붙하는 압축 프롬프트

```
키보드와 마우스/터치로 연주하는 "스틸 텅드럼(steel tongue drum) 웹앱"을 외부 라이브러리·빌드·이미지 없이 단일 index.html(HTML+CSS+Vanilla JS+인라인 SVG)로 만들어줘. 사운드는 Web Audio API 합성.

[화면 흐름]
- 화면1(선택): "11키 텅드럼"(C4–F5·1.5옥타브)와 "15키 텅드럼"(F3–F5·2옥타브) 카드 2개. 각 카드에 해당 드럼의 미니 SVG 미리보기(11키=브론즈, 15키=스틸틸). hover 부드러운 확대, 클릭 ripple + 페이드 전환으로 선택값(11|15) 전달.
- 화면2(연주): 선택한 드럼을 위에서 본 원형 SVG로 화면 중앙에 잘림 없이 최대 크기 표시. "← 다시 선택", 볼륨 슬라이더, 최근음 배지, 키 안내 패널.

[그래픽 — SVG 원형 드럼]
- viewBox 0 0 100 100, 중심(50,52). 본체는 메탈 radial-gradient 원반(circle r=48)+얇은 림. 텅 N개는 각 <g class="tongue" role="button" tabindex="0">로, 중심을 향해 누운 둥근 막대(rect rx 또는 ellipse, rotate로 방사 정렬). 낮은 음일수록 길고 크게(맨 아래=가장 낮음·가장 큰 텅). 외부 이미지 없이 전부 SVG. 텅 자체가 클릭 버튼이라 클릭영역과 시각이 정확히 일치.

[텅 배치(원형)]
- order 배열의 i번째를 원 둘레에 균등 배치: ang=π/2 + i*2π/N (i=0이 6시·맨 아래), x=cx+R cos(ang), y=cy+R sin(ang), 중심 향해 rotate. cx=50,cy=52,R=30은 시작점(시각 미세조정 가능).
- 11키 둘레순서: C4 E4 G4 B4 D5 [F5=맨위] E5 C5 A4 F4 D4
- 15키 둘레순서: F3 A3 C4 E4 G4 B4 D5 [F5=맨위] E5 C5 A4 F4 D4 B3 G3
- (양옆 모두 위로 갈수록 높아지는 대칭 배치)

[음 데이터 — C장조]
- 11키(11음): C4 D4 E4 F4 G4 A4 B4 C5 D5 E5 F5  (낮은음 없음)
- 15키(15음): F3 G3 A3 B3 + 위 11음            (F3~F5, 2옥타브)
- 주파수: 440*2^((midi-69)/12). MIDI: F3=53,G3=55,A3=57,B3=59, C4=60..B4=71, C5=72,D5=74,E5=76,F5=77.

[키 입력 규칙 — 핵심]
- 숫자 1~7 = 계이름(1=도C,2=레D,3=미E,4=파F,5=솔G,6=라A,7=시B). 수식키=옥타브. 같은 숫자=같은 계이름(⌃4=F3,4=F4,⇧4=F5).
- 단독:1~7=C4~B4 / Shift:1~4=C5~F5(⇧5~0 무음) / Ctrl(15키만):4~7=F3~B3(그 외 무음). 11키는 Ctrl 전부 무음.
- 반드시 event.code(Digit1~0 + Numpad1~0)와 shiftKey/ctrlKey로 판별(Shift+1은 key가 "!"). Alt/Meta 또는 Ctrl+Shift 동시 → 무음. 8/9/0 단독 무음. event.repeat 무시. 매핑된 키는 preventDefault. input/textarea 포커스 시 연주 안 함.
- 순수함수 resolveShortcut({code,shiftKey,ctrlKey,altKey,metaKey},"11"|"15")로 구현+테스트: Digit1→C4, Digit8→null, Shift+Digit4→F5, Shift+Digit5→null, Ctrl+Digit4(11)→null, Ctrl+Digit4(15)→F3, Ctrl+Digit7(15)→B3, Ctrl+Shift+Digit4→null, Alt/Meta+Digit1→null, Numpad1→C4.

[사운드 — 텅드럼 음색]
- AudioContext 1회 생성·재사용, 첫 입력에 ctx.resume(). 마스터 게인+볼륨(0~1). 음=기본음+약한 비배음 파셜(2.0x,2.76x,5.18x), 둥근 어택(~12ms)+긴 지수 감쇠(낮을수록 길게 2.2~4.6s), 감쇠하며 닫히는 로우패스. 폴리포니, keyup에서 소리 끊지 말고 자연 감쇠. 권장: 합성 임펄스(노이즈*decay)로 ConvolverNode 리버브(wet~0.25)로 명상적 잔향.

[상호작용/접근성/반응형]
- 누르면 해당 텅에 앰버 글로우+부드러운 물결(드럼 본체는 변형 금지), 최근음 배지 표시, aria-live로 음 이름 갱신. role=button + aria-label(계이름+음명+단축키). 포커스 링. prefers-reduced-motion 반영. 원형 드럼은 어떤 화면에서도 중앙에 잘림 없이 최대 크기.

색: 배경 딥틸→인디고 그라데이션(#0E2A2E→#13203A), 11키 브론즈(#B5772E), 15키 스틸틸(#3E6E78), 글로우 앰버(#FFD27A), 텍스트 크림(#F2ECE0). 차분하고 명상적인 톤, 잔잔한 움직임. 완료 후 주요 구현/실행법/체크리스트를 요약해줘.
```

## 부록 B. (선택) 실물 사진 오버레이 방식

SVG 대신 **실제 텅드럼 사진** 위에 투명 핫스팟을 얹고 싶다면:

- `assets/drum-11.png`, `assets/drum-15.png`(위에서 본 사진)을 준비하고, 사진과 **같은 비율의 정사각 래퍼**(`aspect-ratio:1/1`) 안에 `object-fit:contain`으로 무왜곡 표시.
- 그 위에 `.tongue-layer`(`position:absolute; inset:0;`)를 두고, **12번 `layoutTongues`의 백분율 좌표**로 투명 버튼(텅 핫스팟)을 배치.
- 사진의 실제 텅 위치와 핫스팟이 어긋나면 `cx,cy,R,len,wid`만 조정. **키보드 연주가 1차**라 픽셀 1:1 정합이 아니어도 동작에는 문제 없음.
- 단점: 사진 준비 필요, 정렬이 근사치. **기본 권장은 SVG**(정확·선명·이미지 불필요).
